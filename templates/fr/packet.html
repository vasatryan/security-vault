    Qu’est-ce que les tests d’intrusion?

    Les tests d'intrusion (ou pen testing) sont un exercice de sécurité dans lequel un expert en cybersécurité tente de trouver et d'exploiter les vulnérabilités d'un système informatique. Le but de cette attaque simulée est d’identifier les points faibles des défenses d’un système dont les attaquants pourraient exploiter. C'est comme si une banque engageait quelqu'un pour se déguiser en cambrioleur et tenter de s'introduire par effraction dans son immeuble et d'accéder au coffre-fort. Si le « cambrioleur » réussit et pénètre dans la banque ou dans le coffre-fort, la banque obtiendra des informations précieuses sur la manière dont elle doit renforcer ses mesures de sécurité.

    Qui effectue les tests d'intrusion?

    Il est préférable de faire effectuer un test d'intrusion par une personne ayant peu ou pas de connaissances préalables sur la manière dont le système est sécurisé, car elle pourra peut-être révéler les angles morts manqués par les développeurs qui ont construit le système. C’est pour cette raison que des sous-traitants extérieurs sont généralement sollicités pour effectuer les tests. Ces sous-traitants sont souvent qualifiés de « hackers éthiques » car ils sont embauchés pour pirater un système avec autorisation et dans le but d’accroître la sécurité. De nombreux hackers éthiques sont des développeurs expérimentés titulaires de diplômes supérieurs et d’une certification pour les tests d’intrusion. D’un autre côté, certains des meilleurs hackers éthiques sont des autodidactes. En fait, certains sont des pirates informatiques criminels réformés qui utilisent désormais leur expertise pour aider à corriger les failles de sécurité plutôt que de les exploiter. Le meilleur candidat pour réaliser un pen test peut varier considérablement en fonction de l’entreprise cible et du type de pen test qu’elle souhaite lancer.

    Quels sont les types de tests d’intrusion?

    Pen-test en boîte ouverte: Lors d’un test en boîte ouverte, le pirate informatique recevra à l’avance certaines informations concernant les informations de sécurité de l’entreprise cible.

    Pen-test en boîte fermée: Également connu sous le nom de test « en simple aveugle », il s’agit d’un test dans lequel le pirate informatique ne reçoit aucune information de base autre que le nom de l’entreprise cible.

    Test d'intrusion secret: Également connu sous le nom de test d'intrusion « en double aveugle », il s'agit d'une situation dans laquelle presque personne dans l'entreprise n'est au courant de l'exécution du test d'intrusion, y compris les professionnels de l'informatique et de la sécurité qui répondront à l'attaque. Pour les tests secrets, il est particulièrement important que le pirate informatique ait préalablement écrit la portée et les autres détails du test afin d’éviter tout problème avec les forces de l’ordre.

    Pen-test externe: Lors d’un test externe, le pirate informatique éthique affronte la technologie externe de l’entreprise, comme son site Web et ses serveurs de réseau externes. Dans certains cas, le pirate informatique peut même ne pas être autorisé à pénétrer dans les locaux de l’entreprise. Cela peut impliquer de mener l’attaque à partir d’un endroit éloigné ou d’effectuer le test à partir d’un camion ou d’une camionnette garé à proximité.

    Pen-test interne: Lors d’un test interne, le hacker éthique effectue le test depuis le réseau interne de l’entreprise. Ce type de test est utile pour déterminer l’ampleur des dommages qu’un employé mécontent peut causer derrière le pare-feu de l’entreprise.
